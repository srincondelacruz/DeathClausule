# Práctica Final: Desarrollo de Aplicación SaaS con IA Generativa

Esta práctica final consiste en el desarrollo de una **aplicación SaaS (Software as a Service) completa** que integre procesos de **IA Generativa**. Aplicando todos los conocimientos adquiridos durante el curso, deberás crear una aplicación full-stack con una temática completamente libre que pueda ser interesante para el mercado real (como si se tratara del lanzamiento de una startup).

**Restricción importante:** Está **PROHIBIDO** utilizar agentes de IA. La aplicación debe basarse en procesos de IA Generativa (generación de texto, imágenes, embeddings, RAG, etc.) pero NO en sistemas multi-agente.

**Modalidad de trabajo:** Parejas o tríos.

**Tiempo disponible:** 4-5 días.

---

## Descripción del Proyecto

Tu equipo debe conceptualizar, diseñar y desarrollar una aplicación SaaS completa que resuelva un problema real y tenga potencial comercial. La aplicación debe:

- **Ser full-stack:** Incluir tanto backend como frontend funcionales.
- **Integrar IA Generativa:** Incorporar al menos un proceso de IA Generativa que aporte valor real a la aplicación (no puede ser un simple chatbot básico).
- **Tener enfoque SaaS:** Diseñarse como un servicio que usuarios reales podrían consumir.
- **Ser viable comercialmente:** La idea debe tener sentido de negocio y resolver una necesidad real del mercado.

### Ejemplos de ideas (solo como inspiración):
- Generador de contenido personalizado para redes sociales
- Asistente de escritura técnica/académica con RAG
- Plataforma de análisis de sentimientos para feedback de clientes
- Generador de diseños/mockups a partir de descripciones
- Sistema de recomendaciones personalizadas con embeddings
- Herramienta de optimización de CVs y cartas de presentación
- Plataforma de generación de contenido educativo adaptativo

La temática es **completamente libre**, pero debe demostrar aplicación práctica de IA Generativa.

---

## Criterios de Calificación

La evaluación del proyecto se distribuye de la siguiente manera:

- **Aspecto Técnico (75%):** Calidad del código, arquitectura, implementación de IA, robustez técnica, escalabilidad, documentación técnica.
- **Aspecto Comercial (25%):** Viabilidad de negocio, presentación de ventas, propuesta de valor, potencial de mercado.

---

## Presentación del Proyecto

Cada equipo deberá realizar una presentación completa que incluye:

### 1. Presentación de Ventas (10 minutos)
- **Pitch comercial:** Presenta tu producto como si estuvieras buscando inversión o clientes.
- **Problema y solución:** ¿Qué problema resuelve tu aplicación?
- **Propuesta de valor:** ¿Por qué alguien pagaría por tu servicio?
- **Mercado objetivo:** ¿A quién va dirigido?
- **Modelo de negocio:** ¿Cómo generaría ingresos?
- **Diferenciación:** ¿Qué hace única a tu solución?

### 2. Demo en Vivo (5 minutos)
- **Demostración funcional:** Muestra las características principales de la aplicación en funcionamiento.
- **Flujo de usuario:** Recorre un caso de uso real de principio a fin.
- **Integración IA:** Demuestra claramente cómo la IA aporta valor a la experiencia.

**Tiempo total de presentación:** 15 minutos por equipo.

## Entregables

### 1. Repositorio de GitHub

Debes entregar un repositorio público de GitHub que incluya:

#### README.md completo con:
- **Descripción del proyecto:** ¿Qué hace la aplicación?
- **Tecnologías utilizadas:** Stack tecnológico (frameworks, librerías, servicios cloud, modelos IA, etc.)
- **Arquitectura técnica:** Diagrama o explicación de la arquitectura del sistema
- **Descripción de la integración de IA:** ¿Qué proceso de IA Generativa se utiliza y cómo?
- **Funcionalidades principales:** Lista de características implementadas
- **Tutorial de despliegue local:** Instrucciones paso a paso para ejecutar la aplicación en local (formato open source)
  - Requisitos previos (Node.js, Python, variables de entorno, etc.)
  - Instalación de dependencias
  - Configuración (API keys, bases de datos, etc.)
  - Comandos para ejecutar backend y frontend
  - URLs de acceso
- **Autores:** Nombres de los miembros del equipo

#### Código fuente:
- **Organizado y estructurado:** Carpetas claras para backend, frontend, configuración, etc.
- **Comentado:** Código legible con comentarios donde sea necesario
- **Buenas prácticas:** Seguir convenciones del lenguaje/framework utilizado

#### Otros archivos:
- `.gitignore` apropiado
- `requirements.txt` o `package.json` con dependencias
- `.env.example` para mostrar variables de entorno necesarias (sin credenciales reales)

### 2. Presentación

- **Slides de la presentación de ventas:** Formato PDF o PowerPoint

## Requisitos Mínimos del Entregable

Para que el proyecto sea válido, debe cumplir **obligatoriamente** con:

1. **Aplicación Full-Stack Funcional:**
   - Backend operativo con endpoints/servicios funcionales
   - Frontend con interfaz de usuario accesible y usable
   - Integración correcta entre frontend y backend

2. **Integración de IA Generativa:**
   - Al menos un proceso claro de IA Generativa (generación de texto, embeddings, RAG, fine-tuning, generación de imágenes, etc.)
   - La IA debe aportar valor real a la aplicación, no ser decorativa
   - **Prohibido** el uso de sistemas de agentes

3. **Repositorio GitHub Completo:**
   - README técnico detallado
   - Tutorial de despliegue local funcional y verificado
   - Código fuente completo y ejecutable

4. **Presentación Completa:**
   - Pitch de ventas (10 min)
   - Demo funcional en vivo (5 min)
